Стартап программно-аппаратного комплекса на базе решений Dell EMC / 2018

Внедрение распределенной системы резервного копирования / 2016

Роль: Ведущий инженер проекта
Регион: Пермский край
Область деятельности Заказчика: Топливно-энергетический комплекс

В рамках проекта была разработана архитектура решения и выполнено внедрение разработанного решения в инфраструктура Заказчика.

СРК внедрялась с целью:
  1. Повышения устойчивости информационной среды Заказчика к факторам, вызывающим потерю данных (аппаратным сбоям, ошибкам программного обеспечения и неправильным действиям персонала);
  2. Репликации РК на удаленную площадку Заказчика.
Разработанное решение позволило обеспечить защиту критических данных Заказчика, а также передачу дедуплицированных данных на удаленную площадку Заказчика с их последующим дуплицированием и сохранением. Компоненты решения:
  1. Сервер СРК - Veritas Backup Exec 16 (с Deduplication Option);
  2. Агенты СРК - Veritas Backup Exec 16 Agent.
В решении использовалось существующее оборудование Заказчика (серверы, СХД, сети хранения и передачи данных).

Модернизация СРК / 2016

Регион: Пермский край
Область деятельности Заказчика: Энергетика

  • Ленточные библиотеки - IBM TS3100/3200.
Состав ПО проекта:
  • Сервер резервного копирования - IBM Spectrum Protect 7.1 (ранее IBM Tivoli Storage Manager);
  • Сервер управления и мониторинга - IBM Tivoli Storage Manager Administration Center;
  • Агенты резервного копирования TSM (Data Protection for Microsoft SQL Server, Data Protection for Oracle, Tivoli Storage Manager for Virtual Environments, Data Protection for Exchange);

Внедрение СРК Microsoft System Center DPM / 2015

Роль: Инженер проекта
Регион: Пермский край (удаленно)
Область деятельности Заказчика: Финансово-кредитная организация

В рамках проекта была разработана архитектура решения и выполнено внедрение разработанного решения в инфраструктуру Заказчика.

СРК внедрялась с целью:
  1. Повышения устойчивости информационной системы Заказчика к аппаратным, программным сбоям и ошибкам персонала;
  2. Повышения длительности и уровня сохранности данных;
  3. Обеспечения возможности восстановления информационных систем и отдельных ресурсов;
  4. Снижения длительности плановых (окна резервного копирования) и внеплановых (восстановление после сбоев) простоев ИС Заказчика;
  5. Улучшения управляемости ИС Заказчика за счет централизации управления резервным копированием в многоплатформенной среде;
  6. Обеспечения минимизации операционных рисков потери данных, содержащихся в ИС Заказчика, рисков утраты контроля за имеющейся информацией.
В решении использовалось существующее оборудование Заказчика (серверы, СХД, сети хранения и передачи данных).
Разработанное решение позволило обеспечить защиту критических данных Заказчика.
Также в рамках проекта реализована циклическая защита СРК - когда один сервер DPM защищен другим сервером DPM и наоборот.

Внедрение СРК (региональные отделения) / 2013

Роль: Инженер проекта
Регион: Пермский край
Область деятельности Заказчика: Энергетика

Проводилось проектирование и внедрение системы автоматизированного резервного копирования.

В рамках проекта решались следующие задачи:
  1. Повышение устойчивости информационной среды Заказчика к факторам, вызывающим потерю данных (аппаратным сбоям, ошибкам программного обеспечения и неправильным действиям персонала);
  2. Повышение управляемости информационной среды Заказчика путем централизации управления резервным копированием в распределенной среде;
  3. Снижение длительности плановых (окна резервного копирования) и внеплановых (восстановление после сбоев) простоев ИС Заказчика;
  4. Унификация и стандартизация технологии резервного копирования данных, применение единой технологии СРК, аппаратного и программного обеспечения на региональных площадках.
Компоненты решения:
  • Сервер резервного копирования - IBM Tivoli Storage Manager;
  • Сервер управления и мониторинга - IBM Tivoli Storage Manager Administration Center;
  • Агенты резервного копирования TSM (Data Protection for Microsoft SQL Server,Data Protection for Oracle, Tivoli Storage Manager for Virtual Environments, Data Protection for Exchange);
  • SAN-коммутаторы - Brocade 300 Switch;
  • Виртуальная ленточная библиотека - EMC Data Domain 630;
  • Ленточные библиотеки - IBM TS3100/3200.
Для обеспечения защиты самой СРК использовался Disaster Recovery Manager (DRM), подготовлен план восстановления и обеспечено РК критических данных самой СРК на удаленные хранилища.

Подбор коммерческого ЦОД / 2013

Роль: Ведущий инженер проекта
Регион: Московская область (удаленно)
Область деятельности Заказчика: Лабораторная медицина

Для перевода части сервисов Заказчика на удаленную площадку выполнялся подбор коммерческого ЦОД.

В рамках проекта решались следующие задачи:
  1. Выполнение сравнительного анализа хостинг-провайдеров, отвечающих требованиям Заказчика:
    • Предоставление в рамках хостинг-провайдера услуг:
      • IaaS (инфраструктура как сервис);
      • Аренда оборудования;
      • Размещение оборудования Заказчика (colocation).
    • Площадка хостинг-провайдера должна находиться в России;
    • Возможность составления и подписания дополнительных соглашений по ИБ;
    • Компенсация простоев;
    • т.д.
  2. Миграция (перенос) части сервисов Заказчика в выбранный коммерческий ЦОД:
    • Cервер службы каталогов - OpenLDAP;
    • DNS серверы - BIND;
    • VPN cервер - OpenVPN;
    • Почтовый сервер - связка (Exim + Dovecot + SquirrelMail).
